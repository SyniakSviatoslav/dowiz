# DeliveryOS — Реєстр петель

Джерело істини про наявні петлі. Картки — `loops/<id>.yaml`. Звіти — `loops/reports/`. Пам'ять — `loops/memory/`.
Статуси: **CERTIFIED** (пройшла M1–M11) · **DRAFT** (будується) · **REJECTED** (впала верифікацію) · **DEPRECATED**.
Машинний реєстр роутера `loops/runs/registry.json` — **ПОХІДНИЙ** від цієї таблиці: `node scripts/loops-registry-sync.mjs` (P2 2026-07-02; до цього роутер бачив 2 з 16 петель). `docs/agents/loops/REGISTRY.md` — історичний seed, не джерело.

**Intake (Loop Selection Router).** Над усім — роутер (`tools/loop-harness/src/router.ts`, спец `docs/operating-model/loop-selection-router-v1.md`) вирішує на КОЖНІЙ команді: DIRECT (типове — без петлі) · RUN наявну (`runs/registry.json`) · BUILD нову (loop-builder) · BOUNCE (нема метрики). Routes only — не переписує Class A/B / admissibility / security carve-out. Enforcement = `UserPromptSubmit` hook (`tools/loop-harness/router-hook.sh`, оператор вмикає в .claude) + CLAUDE.md директива.

**Harness (обов'язково для КОЖНОЇ петлі).** Жодної петлі поза харнесом. На фініші (success/stall/abort) петля ЗАВЖДИ емітить §5 LOOP REPORT (повністю в термінал) через `tools/loop-harness` finalize, який міряє git+session-токени+eco і пише незнищенно/lossless у `loops/runs/`. Дизайн — `docs/operating-model/living-loop-system-v3.md`; вузол `harness:` у картці. Телеметрія токенів/агентів/скілів ← session JSONL (те, що читає codeburn). Реалізовано §10 кроки 1–3 + колектори; відкладено §4/§6-recall/§8.

## sandbox-swarm-gate (мета-петля · Sandbox-Swarm-Gate)
Формалізує робочий патерн цієї сесії (worktree-ізольовані Agent-лейни + strong-model review-гейт + merge-on-green) у переюзабельну петлю. **Sandbox** = throwaway git worktree на лейн (`.claude/worktrees/ssg-<lane>`, без prod/secret/main-reach; ітерація всередині НЕОБМЕЖЕНА). **Swarm** = concurrent `Agent(isolation:"worktree")` + `Workflow` (doers=sonnet, reasoning=fable, review=opus). **Gate** = review/audit-межа, де живе справжній quality+safety+**ETHICS** гейт над кожним sandbox-diff ПЕРЕД merge; reject → назад у sandbox. Повна архітектура: **`docs/design/harness/SANDBOX-SWARM-GATE.md`**. Скаффолд-аід: **`scripts/sandbox-swarm-gate.mjs`** (dry-run default; НЕ мерджить, НЕ вимикає ethics-гейт). Картка-DRAFT: `loops/sandbox-swarm-gate.yaml` (чекає loop-architect cert).

- **4-умовний тест (Class):** (1) повторюється? ✅ кожен мульти-лейновий білд/аудит/per-item transform. (2) чітке DoD + машинна верифікація? ✅ гейт-рубрик A/B/C (`scripts/sandbox-swarm-gate.mjs checklist`). (3) бюджет токенів? ✅ рой обмежений лейнами + harness eco-телеметрія. (4) інструменти зробити І верифікувати? ✅ Agent(isolation:worktree)/Workflow (do) + invariant-guardian/security-sentinel/pnpm/Playwright (verify). Усі «так» → це петля.
- **DoD:** кожен диспатчнутий лейн MERGED (гейт A+B+C зелений + red-line human-approval присутній) або явно ABANDONED; 0 sandbox un-reviewed; §5 LOOP REPORT емітнуто; qualified-зміна → reflection (`docs/reflections/INBOX/`) + `REGRESSION-LEDGER.md` guardrail.
- **Verification:** A **Quality** — `pnpm typecheck`+`pnpm build` зелені на лейні · Mandatory-Proof (UI→Playwright toBeVisible/toContainText; API→request.*) · red→green guardrail для bugfix · no-false-green (10 класів). B **Safety** — invariant-guardian VERDICT: PASS · security-sentinel VERDICT: PASS · red-line diff → standing human-approval присутній · protect-paths ре-ранано на merge-diff. C **ETHICS** — Charter зелена (тверда, незнімна; жоден green не перекриває; ethics-red → STOP + людина). Будь-який RED → REJECT назад у той самий sandbox.
- **🔴 Незнімна межа:** «unrestricted sandbox» = НЕОБМЕЖЕНА ІТЕРАЦІЯ КОДУ у throwaway-worktree без real-world-reach — НЕ зняття Ethics Charter (enforced на гейті) і НЕ auto-merge red-lines (auth/money/RLS/PII/secrets/migrations) без гейта+human-approval. Guards РЕЛОКОВАНІ на гейт, не видалені. **Enable-умова (design §9 pairing rule):** sandbox-relaxation хуків (`protect-paths.sh`/`serious-gate.sh`/`red-line-doubt-gate.sh`, keyed на `/.claude/worktrees/<lane>/`) застосовувати ЛИШЕ коли гейт живий — інакше це видалення guard'а. `.claude`/`.github` лишаються hard-blocked навіть у sandbox.
- **DRAFT-гейт:** 🔴 не диспатчиться, доки `/build-verify-loop verify sandbox-swarm-gate` не поставить CERTIFIED (loop-architect M1–M11).

## skill-evolution (мета-петля · Skill Self-Evolution — universal rule)
Розширює self-improvement ратчет ТРЕТІМ виходом: **SKILL** (переюзабельна процедура — SKILL.md: опис-що-тригерить + кроки + checkable DoD) поряд із **guardrail** (authority) і **lesson** (advisory). Універсальне правило (директива оператора 2026-07-03): кожен агент, кожна задача — **find → use → expand → create** скіли з досвіду/задач/пам'яті. **Sibling of sandbox-swarm-gate** — REUSE'ить той самий sandbox+gate субстрат: новий/розширений скіл ітерується у SSG-worktree, гейт мерджить. Повна архітектура + rubric + decision-tree: **`docs/design/harness/SKILL-EVOLUTION.md`**. Драфт-скіл цієї сесії: `docs/design/harness/proposed-skills/max-lanes-orchestration/` (quick_validate ✅). Картка-DRAFT: `loops/skill-evolution.yaml` (чекає loop-architect cert).

- **Routing (за формою, не за зручністю):** *процедура* → skill · *баг, що ловить машина* → guardrail · *точкове попередження на місці* → lesson. НЕ взаємовиключні — багата рекуренція дає кілька одразу. Skill НІКОЛИ не замінює guardrail на red-line (signals inform, deterministic artifacts decide).
- **Promotion bar (коли процес стає скілом):** усі три — **рекурувало ≥2×** AND **стабільні кроки** AND **checkable DoD** (та сама планка, що робить процес петлею). Одне-два з трьох → lesson/reflection, не skill. Не створювати скіл на one-off — переповнений store розмиває триґер (bias-to-prune, як для lessons).
- **4-умовний тест (Class):** (1) повторюється? ✅ будь-яка процедура, яку агенти re-derive'ять щосесії (напр. lane-collision клас — 2 рефлексії). (2) чітке DoD + машинна верифікація? ✅ quick_validate + skill-creator description-eval (should/should-not triggers) + steps-work. (3) бюджет? ✅ ітерація у sandbox-worktree + eco-телеметрія. (4) інструменти зробити І верифікувати? ✅ skill-creator/find-skills (do) + quick_validate/eval/security-sentinel/ethics-charter (verify). Усі «так» → це петля.
- **DoD:** FIND перед будь-яким CREATE; процедура пройшла bar АБО коректно зупинилась на lesson/guardrail; новий/розширений скіл пройшов гейт (A+B+C proof-credited) або явно ABANDONED; скіл DRAFTED у `docs/design/harness/proposed-skills/` (НЕ self-install у protected `.claude/skills/` — оператор ставить); curation-pass (distill·challenge·promote·prune — store не росте); §5 LOOP REPORT; qualified-зміна → reflection.
- **Verification:** гейт-рубрик (SKILL-EVOLUTION.md §7, reuse SANDBOX-SWARM-GATE §4): A **Quality** — quick_validate.py зелений (frontmatter/kebab/≤1024/no-angle-brackets) + description-eval тригерить правильно на should/should-not + steps працюють на ≥1 тесті + checkable DoD (не aspirational) + не дублікат (EXPAND замість). B **Safety** — нема malware/PII/secret/surprise + жодного gate-weakening кроку + security-sentinel PASS + нема protected-zone/auto-merge інструкції. C **ETHICS** — Charter зелена (тверда, незнімна). Будь-який RED → REJECT назад у sandbox.
- **🔴 Незнімна межа:** скіл — це executable capability, що піде у кожного майбутнього агента → той самий гейт, що й код; ETHICS Charter enforced на гейті (не знімається у sandbox); `.claude`/`.github` hard-blocked навіть у sandbox (скіл не може переписати гейт/етику, що його судить, ані self-install'нутись); skill НІКОЛИ не мінтить authority, якої не має (не замінює guardrail, не оминає approval).
- **DRAFT-гейт:** 🔴 не диспатчиться, доки `/build-verify-loop verify skill-evolution` не поставить CERTIFIED (loop-architect M1–M11).
