# /compliance — єдине джерело істини комплаєнсу DeliveryOS

Тримає всі privacy/комплаєнс-артефакти живими, версіонованими, під review.
Controller у всіх документах = **[SHPK назва ____]** (Албанія). Рамка: Закон 124/2024 (GDPR-aligned).
Це робочі документи, не юрпорада; регулятор-фейсинг фінал звіряй з юристом (за потреби — албанською/англ.).

## Три типи документів і де вони ЖИВУТЬ
- **Публічні політики** (privacy, terms) → публікуються на dowiz.org із версією+датою; тут лише ЗНІМКИ версій (policies/).
- **Підписані контракти** (founders, IP, DPA) → підписані PDF у захищеному сховищі (НЕ в git); тут лише шаблони + статус (contracts/).
- **Внутрішні записи** (RoPA, DPIA, data-map, інваріанти) → живуть тут у git, міняються через PR, віддаються Commissioner на запит.
- **Згода** → це РЯДКИ В БД, не файли (patterns access_requests.consent_at + privacy_version). Див. consent-records.md.

## Карта файлів
- data-map.md — інвентар PII + controller/processor (джерело істини)
- RoPA.md — реєстр обробки (Art 30 / 124/2024)
- subprocessors.md — реєстр суб-процесорів
- privacy-invariants.md — кодовані правила + чекліст CI-гейта
- dpia/ — оцінки впливу (high-risk обробка)
- policies/ — знімки опублікованих політик
- contracts/ — шаблони + статус підписаних контрактів
- consent-records.md — де й як зберігається згода (БД)

## Тригери оновлення
| Подія | Оновити |
|---|---|
| Нова PII-колонка/поле | data-map.md + RoPA.md (+ перевір erasure-покриття) |
| Новий зовнішній виклик/інтеграція | subprocessors.md (+ DPA якщо несе PII) |
| Нова/змінена фіча обробки | RoPA.md + перевір потребу DPIA |
| High-risk обробка (локація/біометрія/масштаб) | новий dpia/*.md |
| Зміна практик чи суб-процесорів | нова версія політики (policies/) + re-publish |
| Зміна правил | privacy-invariants.md |
| Нова точка збору згоди | consent-records.md + ts+version у таблицю |

## CI-гейт (перед merge)
Агент/CI дифить схему й нові seams проти privacy-invariants.md і блокує merge, якщо:
- нова PII-колонка без controller+підстави+ретенції+erasure-покриття;
- новий зовнішній виклик не в subprocessors.md (або несе сирий PII без DPA);
- сирий PII у логах/чергах/шині;
- high-risk поле без DPIA.
