## PURPOSE
DeliveryOS ("dowiz") — a 0%-commission restaurant ordering/delivery platform (Albania-first). Three surfaces: public storefront `/s/:slug` (SSR menu, order, track), owner `/admin/*` (menu/orders/couriers/settlements), courier `/courier/*` (shifts/assignments/deliver). Goal: open-source under AGPLv3 (gated on a secrets-history scrub + EUTM). Prod = dowiz.fly.dev; staging = dowiz-staging.fly.dev.

## STACK
Being REBUILT code-only (schema unchanged) from Node/TS (fastify + preact SSR) to **Rust** (axum + tokio + tower + sqlx compile-checked, `rebuild/crates/{api,domain}`) + **Astro 5 / Svelte 5 islands** frontend. DB = Supabase Postgres with **RLS FORCE + NOBYPASSRLS** target (currently BYPASSRLS pre-B3-flip). Fly.io (fra) + Cloudflare R2, NO Redis, **integer minor-unit money** (ADR-0005, `domain::Lek`), **forward-only migrations** (`packages/db/migrations/`). Token economy: VSA codec (`tools/vsa`), codebase-memory graph (this), repowise. OpenAPI 3.1 SSOT via utoipa.

## ARCHITECTURE
**Strangler-by-surface cutover** S1..S10 = storefront-read / auth / catalog / media / orders+money / realtime-WS / courier / jobs / GDPR / platform-admin. Node stays the single public ingress; a front-door hook (`apps/api/src/lib/cutover/`) resolves each request's surface via a 236-route template matcher, reads Postgres `cutover_flags`, and either serves Node or streams to the internal Rust app over Fly flycast (port 80). Per-surface flip = one flag UPDATE (reversible). A generated **per-route NODE-KEEP gate** (`node-keep.generated.ts`, from `scripts/rebuild-cutover/gen-node-keep.mjs`) keeps mapped-but-unported routes on Node even when their surface is flipped. Bot UAs on `/s/:slug` → Rust JSON-LD; humans → Astro. As of 2026-07-05: 9/10 surfaces live-proven on staging, full order lifecycle L0-L7+L11 green. Prod flip is operator-gated (S5-money + S9-GDPR = explicit confirm; Phase-D decommission owner+date HARD gate).

## PATTERNS
- **THE cutover SQL bug class** (regression-ledger #77): passes as a psql literal / text-render but FAILS as a sqlx bind/decode. Every enum column needs `::<enumtype>` on a BOUND write AND `::text` on a String READ; `numeric`→f64 needs `::float8`; `int4`→i64 needs `::bigint`. The real fix is the CI live-PG suite (`docs/design/ci-rust-live-pg/`) running the `#[ignore]` tests against real Postgres — a psql-literal probe is a FALSE witness; reproduce with a BOUND param (`PREPARE t(text) AS ...; EXECUTE t('x')`).
- **RLS tenancy seams**: `db::with_user(app.user_id)` for OWNER writes; `db::with_tenant(app.current_tenant)` for courier/service; anonymous order-create is **GUC-less by design** (anonymous_insert admits on `app_current_user() IS NULL`). FORCE-RLS cross-tenant reads (owner_notification_targets, gdpr claim) go through SECURITY DEFINER fns (pinned search_path). `assert_active_owner_membership` is the FIRST statement in every owner write-tx.
- **Money**: integer minor-units end-to-end; `request_hash` idempotency (`(key,location_id)` UNIQUE + integer-projected canonical hash); order state-machine byte-identical to Node (100-pair exhaustive test); cash-as-proof = exact `cash_amount === total`.
- **Token economy (3 complementary layers)**: VSA compresses DATA payloads (frames, ~34-55%); codebase-memory graph compresses CODE-STRUCTURE navigation (~55% A/B, query don't read files); repowise = WHY-synthesis + semantic search. Wire names camelCase + toISOString + JS-number (integral f64 → no decimal) for Node parity.
- **Ship discipline**: commit → staging deploy → Playwright/live proof (Mandatory Proof Rule). **Probe-and-approve** (operator waived councils 2026-07-05): port → live-probe on staging → approve on green. Every fix → deterministic guardrail (red→green) + regression-ledger row.
