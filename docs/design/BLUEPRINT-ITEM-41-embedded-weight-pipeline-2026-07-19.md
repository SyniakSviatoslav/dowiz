## 1. Scope / goal + non-goals

**Goal.** A generator (extending the `eqc_gen` / `gen_kernel_organs` pipeline) that emits the frozen
quantized pilot weights (item 34 + item 35) as **committed, hand-inspectable Rust source**:
`#[repr(align(64))] static WEIGHTS: [i8; N]`, with an **init-time SHA3-256 self-check** against a
committed golden hash (reusing `event_log::sha3_256`), and **ML-DSA-65 codesign** via
`pq/codesign.rs` for the update-blob shipping mode.

**Non-goals.** NOT the objcopy/`link_section` "Linker Way" — **parked** with a named reopening
trigger (§6). NOT `include_bytes!` (kills compile time, spams the IDE — dialogue part-5 §1). NOT a
`build.rs` (no build machinery — the eqc_gen precedent is generate-once, commit, compile-normally).
NOT a new crate — the zero-dep gate stays untouched (SHA3 and ML-DSA are already in-tree).

## 2. Current-state grounding

- **"Data as code" precedent is generated-committed Rust, not linker sections.** `eqc_gen.rs:1` is
  the canonical shape: "GENERATED by eqc-rs — do not hand-edit," hand-inspectable, diff-reviewable,
  zero build machinery, and a test asserts exact-integer equality against the source law
  (`eqc_gen.rs` money-organ parity). Item 41 is the *same* pattern for weights.
- **The signing half already exists.** `pq/codesign.rs` authenticates update blobs by ML-DSA-65
  against ONE `PinnedRoot` (`:30`), distinguishes `Signed`/`Unsigned` (`:40`, RED gate), refuses
  unsigned (`CodeSignError::Unsigned`, `:49`). Reused verbatim for the weight-update-blob path.
- **The hash half already exists.** `event_log::sha3_256` (`:109`, from-scratch Keccak) is the init
  self-check primitive — no new hash.
- **`#[repr(align(64))]` is genuinely new** (synthesis §1.5 GROUNDED negative: it appears nowhere in
  the workspace) — **flagged as a new surface** for review, not a blocker.

## 3. Implementation plan

1. **Generator emits committed source.** A `tools/eqc-rs`-adjacent binary (the `gen_kernel_organs`
   sibling) takes the frozen, quantized i8 weights and emits a committed Rust file:
   `#[repr(align(64))] pub struct AlignedWeights(pub [i8; N]); pub static WEIGHTS: AlignedWeights = AlignedWeights([ … ]);`
   in the `eqc_gen` "GENERATED — do not hand-edit" style (hand-inspectable, diff-reviewable).
2. **`#[repr(align(64))]` wrapper** — 64-byte alignment so item 39's aligned SIMD loads
   (`vmovaps`-class) never fault and never pay a misalignment penalty (dialogue part-5 §3). A test
   asserts `(&WEIGHTS as *const _ as usize) % 64 == 0`.
3. **Init-time SHA3-256 self-check.** At engine init, compute `event_log::sha3_256(&WEIGHTS.0)` and
   compare to a committed `GOLDEN_WEIGHTS_HASH`. Mismatch → refuse to run (fail-closed) — proves the
   executing weights are exactly the compiled-and-verified ones (dialogue part-5 §2; Q4 mechanism).
4. **Parity test (the eqc_gen discipline).** A test asserts the emitted `static WEIGHTS` equals the
   frozen quantized weights **exactly** — the generator and its output cannot silently diverge (the
   `eqc_gen` exact-integer-equality precedent).
5. **Update-blob shipping via codesign.** When weights ship as an update blob (not embedded in this
   binary), wrap them in a `pq::codesign` `UpdateBlob::Signed` (ML-DSA-65 against the pinned root);
   `apply` refuses `Unsigned`. (For the embedded v1, the *whole kernel binary* is the signed unit;
   codesign is the mechanism for the OTA path.)
6. **Zero-dep gate untouched** — no crate involved; `objcopy`, if ever adopted, is CI-time tooling in
   the Kani class (§6 trigger).

## 4. Required proofs (5-point checklist mapping)

- **1 (oracle):** N/A (data pipeline) — but the **parity test** (emitted static == frozen weights)
  is the exact-equality oracle in the `eqc_gen` sense.
- **3 (differential):** the init self-check is a continuous SHA3 differential vs the golden hash.
- **P7 (planted-fault, load-bearing):** the init self-check demonstrably **FAILS on a tampered byte**
  (red→green); alignment asserted by test.
- **2/4/5:** N/A — no timing surface (init hash is not secret-dependent), no new algorithmic asm
  (SHA3 is item-7-covered), no new Kani target.

## 5. Falsifiable acceptance criteria

1. The generator emits a committed `#[repr(align(64))] static WEIGHTS: [i8; N]` in the `eqc_gen`
   "GENERATED — do not hand-edit" style (hand-inspectable, diff-reviewable).
2. A **parity test** asserts the emitted static **equals the frozen quantized weights exactly**.
   **RED→GREEN:** perturbing one weight in the source vs the golden diverges.
3. The init **SHA3-256 self-check (via `event_log::sha3_256`) FAILS on a tampered byte** (RED→GREEN)
   and passes on the golden.
4. **Alignment asserted by test** (`address % 64 == 0`).
5. The update-blob path uses `pq/codesign.rs` (ML-DSA-65, refuses `Unsigned` — reuse, not a new
   codesign).
6. The parked objcopy/`link_section` alternative + its **named reopening trigger** is recorded in the
   module doc (slot_arena-format).
7. **Zero-dep gate untouched** — `cargo tree -e no-dev` still resolves to the kernel root alone.
