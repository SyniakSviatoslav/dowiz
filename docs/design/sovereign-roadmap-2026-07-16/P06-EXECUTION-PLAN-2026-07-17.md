## §2 — Remaining (OPERATOR-GATED: requires Phase 3 to close C4b on `mod_l`)

P06's own blueprint §0/§7.9 hard-gates real key_K/key_V signing on Phase 3 closing
**C4b** (`mod_l` side-channel on the Ed25519 path, HIGH). Until C4b is GREEN, the
`Signer` trait MUST stay at `UnsignedSigner` — shipping signed output now would be
a false GREEN (a self-certifying verdict with no real key separation).

Concrete next steps once C4b is GREEN:

1. **Real digest** — replace `digest32` (`git hash-object`, sha1) with sha3-256.
   Either vendor a KAT-gated `sha3` (e.g. `tiny-keccak` with the KECCAK-256 /
   SHA3-256 flag) or shell to `sha3sum` once present on CI hosts. The TLV schema
   (32-byte slots) is already correct; only the primitive swaps.
2. **Verify-only Ed25519/ML-DSA-65** — add a `bebop2` path-dependency (or vendored
   crate) exposing `verify_pq` / `verify_classical` from the idle PQ substrate
   (`pq_dsa`, `hybrid_gate`). Wire a `HybridSigner` implementing `Signer` using the
   K/V anchors from `config/kv-genesis.txt`. NO signing keys are generated by
   ci-truth; it only *verifies* notes produced by the operator-held key_K/key_V.
3. **Note production tooling** — a separate, operator-run `ci-truth v1-sign`
   (or an offline ceremony) that produces the two git notes using the real keys.
   ci-truth itself never holds the signing secret; it only verifies.
4. **Tests** — extend the gate suite with real-signature happy-path (GREEN) and a
   known-bad-signature forgery case (RED), reusing the WIRE-style anchored chain
   pattern from bebop2 `proto-cap`.

## §3 — Why this ordering (Anu check)

P06's value is the *policy contract* that downstream arcs already consume
(hermetic H3 breach-probe verdicts, spectral E3 phase-gate, phases 7/9/10). That
contract is shippable and testable today with zero crypto. The crypto is a
narrow, well-bounded swap behind `Signer` + `digest32`, gated on C4b for a real
side-channel reason — not a "nice to have". Building fake PQ now would violate
AGENTS.md's "never fake crypto" and P06's own precondition; shipping the contract
without it is the honest, high-leverage step.
