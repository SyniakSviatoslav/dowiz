## 1. Faithfulness is out; truthfulness is the criterion

Operator directive: the model's self-report of its own correctness ("faithfulness") is not a
safety signal and is removed from the breaker's input vector entirely. A compromised or
hallucinating model reports fine; the signal is generated by the thing being measured. It fails
the breaker's own admission rule from Synthesis I — signals must be computable *outside* the
monitored agent.

Replacement — **truthfulness**, defined operationally: *given byte-for-byte identical input and
conditions, the model must produce byte-for-byte identical output, checked at different points in
time. Any divergence is a signal.* This is cache semantics: same key + same conditions ⇒ same
value; a cache that returns different values for the same key is corrupt by definition, and no
interpretation of the value's *content* is needed to know it. The check is content-free,
deterministic, and adversary-hostile: a poisoned memory, a swapped weight file, or a hijacked
system prompt all change the function being evaluated, and a replay of a stored (input, output)
pair detects the change without understanding it. **[GROUNDED as a definition; detector efficacy
PROPOSED — see §2 prerequisite.]**

This slots into the existing breaker as a new signal class: **replay probes**. Half-Open canary
probes (already in Synthesis I) become exactly this — stored golden (input, output) pairs
replayed against the quarantined agent; bitwise match required to close.

## 2. Hard prerequisite: batch-invariant inference, or the signal is noise

Research verified this session (honest summary, including the part that hurts):

- The root cause of temp=0 nondeterminism in served LLMs is **not** GPU atomics. It is
  dynamic/continuous batching in serving engines (vLLM, SGLang): reduction-kernel summation order
  varies with concurrent load, floating-point non-associativity turns order into different bit
  patterns, and bit drift in logits flips greedy-decoded tokens. MoE adds a second source: router
  logits drift the same way, changing expert routing. **[GROUNDED]**
- The fix exists: Thinking Machines Lab's `batch_invariant_ops` (Sept 2025) makes
  RMSNorm/matmul/attention invariant to batch composition. Measured: **1000/1000 bitwise-identical
  completions vs. 80 unique completions per 1000 at baseline**. Cost ~1.6–2.1× unoptimized, ~34%
  overhead with SGLang CUDA-graph integration. **[GROUNDED]**
- Consequence stated as a hard prerequisite, not a footnote: on ordinary serving infrastructure,
  **~92% of identical-input runs already diverge with zero poisoning or hallucination present**.
  Without batch-invariant kernels deployed first, the truthfulness signal has a false-positive
  floor near 1.0 and is useless. The deployment order is therefore fixed: batch-invariant kernels
  → verify 1000/1000 locally → *then* arm replay probes. Synthesis I already flagged vLLM as
  non-bitwise-deterministic by default; this section is why that flag is load-bearing rather than
  cosmetic.
- Novelty, honestly assessed: no prior work uses byte-identical-repeat-output as a
  poisoning/hallucination detector. The closest analog, SelfCheckGPT, samples at temp=1 and does
  semantic/NLI diffing — it *expects* variance and mines it; the operator's scheme *forbids*
  variance and treats it as evidence. Opposite assumption, genuinely novel combination, with the
  §2 dependency as its single point of failure. **[GROUNDED negative-result search; novelty claim
  is absence-of-evidence, not proof of absence.]**

## 3. External deterministic oracles: real precedent, corrected scope

PDDL-INSTRUCT (arXiv:2509.13351, MIT CSAIL) uses VAL — a classical PDDL plan validator, not Fast
Downward — as an external deterministic oracle giving categorical per-precondition pass/fail
feedback to an LLM planner. **Correction to an earlier secondhand account:** the paper does *not*
have a scalar violated-precondition count with a formal convergence-to-zero stopping criterion;
the actual stopping rule is a fixed 10–15 iteration budget. No convergence proof exists there, and
none should be cited.

What survives the correction is narrower and still useful: an **external, deterministic,
categorical oracle checking per-precondition validity** is a published, working pattern. That is
precisely the class of the "constraint-violation count" signal already in Synthesis I's vector —
this is precedent-validation of an existing signal, not a new metric. **[GROUNDED, with the
overstatement explicitly retired.]**

## 4. Time as the contraction metric — closing Synthesis I's open Banach question

Synthesis I left open: any generate→evaluate→refine loop is iteration of an operator T; is it a
contraction, with what q, under what metric? Operator's answer: **use time**, via the topology and
chronology layers already live in this repo. Both verified in-tree this session:

- **Topology layer** — `kernel/src/order_machine.rs`: `has_cycle`, cyclomatic number
  μ = |E|−|V|+c, `topological_order`, `reachable`, `spectral_radius` (ρ via power iteration, now
  compile-time-frozen with the identity ρ≈0 ⟺ acyclic ⟺ topo-order-exists enforced in tests).
  **[GROUNDED]**
- **Chronology layer** — `kernel/src/bin/markov_attractor.rs` (native Rust; the old
  `tools/loop-signals/markov_attractor.py` was ported — `tools/loop-signals/check.sh` now pipes
  `transcript_events.py` → the kernel binary), computing over a rolling time-ordered window of
  tool-call outcomes: entropy-rate, escape-mass, Foster-Lyapunov drift, and SLEM + period;
  verdicts HEALTHY / LIMIT_CYCLE / STRANGE_ATTRACTOR, live in this session's own hooks.
  **[GROUNDED]**

Mapping these onto q, with rigor separated from analogy:

**Rigorous within the Markov model.** If the loop's outcome process really is a 1st-order Markov
chain, then: a negative Foster-Lyapunov drift condition implies geometric ergodicity —
convergence to the stationary distribution at a geometric rate, which is *literally* a contraction
statement with the total-variation-to-stationarity playing the role of the metric. SLEM is the
standard bound on that rate: mixing behaves like SLEM^n, so **SLEM plays the role of q for the
chain**. SLEM→1 = vanishing spectral gap = loss of contraction; SLEM near 1 *with period ≥ 2* is a
limit cycle — exactly the detector already shipping. Escape-mass→1 (long-run occupancy of the
progress state) is the statement that the stationary distribution concentrates where we want.
These are theorems, not metaphors — *conditional on the model*.

**The honest gap.** Banach's q lives on the *artifact space* (the thing being refined) under some
metric d; the Markov chain lives on a coarse projection — outcome tokens of tool calls. Two
failure modes of the mapping: (a) **aliasing** — artifact space can orbit while the projection
looks stationary (distinct artifacts, same outcome token), so escape-mass→1 is supporting evidence
for q<1, never proof; (b) the outcome token is not a deterministic function of artifact state
(environment noise enters), so even a detected cycle in the projection is strong evidence, not
proof, of non-convergence upstream. Additionally the 1st-order Markov assumption itself is
unvalidated for this process. **[ANALOGY at the projection boundary — flagged, and measurable: see
step 8 below.]**

**The reframe that makes "time" rigorous.** Time-to-green is not a metric on artifact space (it
fails symmetry and identity-of-indiscernibles), so "Banach with d = time" is the wrong
formalization. But time is exactly a **Lyapunov potential**: take V(state) = expected residual
time to the accepting state. Foster-Lyapunov asks for E[V(next) − V(now)] < −ε — and the drift
statistic the attractor already computes is an empirical estimate of that expectation. So the
operator's instinct lands on the *correct* formal object, just under Lyapunov's name rather than
Banach's: measure successive inter-checkpoint times t₁, t₂, …; if they decay geometrically, the
fitted ratio **q̂ = t_{n+1}/t_n** is an empirical contraction rate, falsifiably estimable per loop
with confidence intervals. Banach is recovered as the special case where V itself contracts
geometrically. **[PROPOSED — the estimator is defined; no q̂ has been measured yet.]**

## 5. Three physics precedents → three design rules (load-bearing citations only)

- **Kepler (conservation → cheap verification).** The second law's Ȧ=const holds for *any*
  central force because it is angular-momentum conservation — a symmetry statement, letting
  trajectory correctness be checked without integrating the equations of motion. Rule: wherever a
  swarm state transition has a conserved quantity, check the invariant, never re-simulate. One
  instance already landed: `TokenBucket::release` is a conservation law for permits (Synthesis I).
  The audit action is to enumerate the others — message in/out counts, event-log monotone
  sequence, budget totals — and feed violations into the existing constraint-violation signal.
  **[GROUNDED principle; enumeration PROPOSED.]**
- **Hooke (every stable equilibrium is linear up close).** k = U″(x₀): near any stable fixed
  point, dynamics are spring-like to first order. This upgrades the EWMA/CUSUM linear-regime
  signals of Synthesis I from "heuristic choice" to "principled first line": near the healthy
  operating point, linear thresholds are *sufficient* by Taylor's theorem, and the
  categorical/nonlinear machinery (replay probes, oracle checks) is the escalation for large drift
  where the linearization breaks. Open and measurable: the radius of validity of the linear regime
  is unknown and sets the escalation threshold. **[GROUNDED principle; radius PROPOSED.]**
- **Frenet-Serret (minimal sufficient statistic).** A curve is determined up to rigid motion by
  two scalars, κ(s) and τ(s) — the representation where everything that isn't shape disappears.
  Design directive for telemetry: log a minimal trajectory statistic (e.g., per-step action-class
  + state-delta digest), not raw high-dimensional state; "two trajectories are the same shape"
  then becomes the equality predicate the §1 chronological replay comparison needs, at a fraction
  of the storage. Flagged honestly: no theorem guarantees a κ,τ-analog exists for discrete agent
  trajectories — this is a search directive for a representation, not a claimed result.
  **[ANALOGY — explicitly unproven; success criterion in step 12.]**

## 6. Code discipline: twelve anti-patterns, one sanctioned god-function

All twelve standard anti-patterns (magic numbers, nested conditionals, long parameter lists,
duplicate code, deep nesting, boolean flags, commented-out code, god functions/objects,
stringly-typed code, callback hell, mutable shared state, vague names) apply as lint-grade
discipline to the swarm codebase — **with exactly one deliberate exception**: the
emergency-recovery / break-glass path (Shamir's-Secret-Sharing key reconstruction, from the
original brainstorm) is *intentionally* a single monolithic function. Reasoning, stated rather
than asserted: the god-object anti-pattern is bad because it is *accidental* complexity sprawl —
many owners, many call sites, unaudited growth. The break-glass path inverts every one of those
properties: it must have exactly one entry point (one thing to gate behind the human
`Killed`-state unlock), one audit surface (a reviewer can hold the entire recovery semantics in
one reading), and zero composition with ordinary code paths (helpers shared with normal flow are
places to hide a backdoor into recovery). Monolithic-by-design here is a security control, not
debt. Everywhere else the twelve rules bind.

Concrete distributed-systems picks (one choice per decision, justified, alternatives rejected):

- **World-Model sharding: geo/spatial shard key + rendezvous (HRW) hashing for placement.** The
  domain is delivery — queries are radius-shaped and spatially local; hash sharding destroys that
  locality and range sharding hotspots on dense cities, so the shard key is a geo-cell prefix.
  Placement uses rendezvous hashing over a consistent-hash ring: HRW is a single pure function
  (auditable, deterministic — matches the repo's determinism-first stance), needs no virtual-node
  tuning, and gives minimal (1/n) key movement on membership change; ring+vnodes buys nothing at
  this shard count. **[PROPOSED]**
- **Steward election: etcd-style TTL lease + fencing token.** Kubernetes-native rejected — this
  repo carries a standing no-K8s decision, and a reconciliation controller is a nondeterministic
  authority in a determinism-first design. Consul adds a second consensus system to operate. The
  lease *semantics* (bounded-TTL leadership, monotone fencing token attached to every Steward
  action so a stale leader's writes are rejectable) are what matter; the event-sourced core's
  monotone log sequence already provides the fencing token, so implement lease state over the
  existing kernel rather than importing a new dependency. **[PROPOSED — semantics fixed,
  implementation unbuilt.]**

   the §2 confound on *our* stack. Prediction from the research: tens of unique outputs; if we get
   1000/1000, the prerequisite is already met and §2's cost estimate is void.
7. **Retrospective SLEM/escape-mass validation (zero new compute).** Historical session
   transcripts (`journal.jsonl` corpus) already exist; run the attractor binary over labeled
   stuck-vs-healthy sessions; report precision/recall of LIMIT_CYCLE / STRANGE_ATTRACTOR verdicts.
   Falsifiable: if SLEM→1 doesn't beat a coin flip on labeled loops, the §4 q-proxy claim is
   downgraded to dead.
8. **q̂ estimator.** Instrument one refine loop with inter-checkpoint times; fit geometric decay;
   publish q̂ ± CI per task class. Simultaneously test the 1st-order Markov assumption (compare
   1st- vs 2nd-order transition likelihoods) — this bounds how much of §4's "rigorous within the
   model" clause actually applies.
9. **Batch-invariant canary.** Deploy `batch_invariant_ops` (or SGLang deterministic mode) on one
   replica; repeat step 6; arm the truthfulness replay-probe signal only on 1000/1000. Record the
   measured throughput cost against the 1.6–2.1×/34% published figures.
10. **Conservation audit (Kepler).** Enumerate candidate conserved quantities across swarm
    transitions (permits, message counts, log monotonicity, budget totals); implement each as an
    invariant check feeding the constraint-violation signal; count how many exist — if <2 beyond

    compression vs raw logs, (b) collision rate — how often distinct-outcome trajectories map to

    back to search.
