# Principle 1 — Mentalism (architecture grounding + compliance audit)

> One of 7 parallel Hermetic-principle passes. Self-contained; a later Fable pass synthesizes all 7.
> Every claim below is `file:line`-cited against the live tree (dowiz `HEAD` on
> `feat/kernel-fsm-graph-analysis`, and `/root/bebop-repo`). No mysticism — the mystical text is
> used only as the *name* of an engineering rule, not as evidence for anything.

The mystical statement ("The All is Mind; the Universe is Mental") is not the deliverable. What is
load-bearing is its one falsifiable engineering claim: **a thing exists as a pattern/idea before it
exists as a manifestation.** This document grounds that as a binding architecture rule for this
codebase, then audits where the codebase keeps the rule and where it breaks it.

---

## 1. The concrete architecture-principle statement

**MENTALISM (this codebase): the specification, the proof, or the decision record is the source of
truth — the code is a *derived artifact* of it. A capability must exist as a checkable idea
(equation, ADR, blueprint done-check) before or alongside the code that manifests it; code without a
recorded idea, and an idea asserted as real without code that manifests it, are both defects.**

This is not imported from the Kybalion. The project arrived at it independently, three times, in
pre-existing rules:

- **Precedent A — "the plan is the spec."** `AGENTS.md:207-210` (Detailed Planning Protocol step 8):
  *"the plan is the spec — deviations get written back into it, never silently diverged from."* The
  same protocol demands *ground-truth-before-design* (`AGENTS.md:171-177`), *falsifiable done-checks,
  not vibes* (`AGENTS.md:191-192`), and *DECART-every-integration-inline-before-the-blueprint-is-done*
  (`AGENTS.md:181-184`). The idea (plan) is authoritative; the implementation is bound to it.
- **Precedent B — VERIFIED-BY-MATH.** The standing rule (`/mnt/volume-fsn1-1/dowiz-memory/verified-by-math-2026-07-07.md`,
  indexed in `MEMORY.md`): *works? provable? falsifiable? → ship RED otherwise.* A capability that
  cannot be stated as a checkable proposition does not ship green. The idea (the falsifiable claim)
  gates the manifestation (the merge).
- **Precedent C — Anu / Ananke doctrine.** `AGENTS.md:234-241`: *"A plan fails Anu when a decision is
  asserted but not derivable from evidence already in front of the agent that made it."* This is
  Mentalism's contrapositive stated as governance: an idea that cannot be re-derived from evidence is
  not a valid idea, so the manifestation built on it is unfounded.

Three further real instances confirm the principle is *implemented*, not just professed:

- **`eqc` — the literal case.** `tools/eqc/README.md:1-16,33-44`: *"One source-of-truth math
  expression → committed, hand-inspectable Rust… The equation is the source; the `.rs` is a derived
  artifact, like an object file from source."* This is Mentalism as a compiler: the Rust *cannot
  exist* until the mathematical idea (a SymPy expression) compiles into it, and `emit_proof_program()`
  emits a program that self-asserts f64 ≈ fixed ≈ SymPy reference at sample points. The emitted
  artifact carries its own proof of fidelity to the idea. Verified live at
  `/root/bebop-repo/rust-core/eqc-proofs/lambda_max_of_d.rs:1-19` — header reads
  `// GENERATED by eqc — self-asserting proof for lambda_max_of_d.` / `exit 0 ⟺ the emitted f64
  (and fixed-point) code matches the SymPy reference.` The idea `lambda_max_of_d = 2*d` is the source
  of truth; the function body `2.0*d` is the shadow it casts.
- **`evals.rs` `MetamorphicGenerator` — instance defined by its relation before its data.**
  `kernel/src/evals.rs:125-297`. Each generator (`spectral_similarity` :147, `kalman_q_scaling` :182,
  `noether_conserving` :210, `recall_constructed` :263) *mints* a benchmark instance whose oracle **is
  a kernel primitive / metamorphic relation** — the correctness of the instance is defined by the
  mathematical relation it must satisfy *before any concrete sampled data exists*. The `MintLog`
  (:71-108) additionally rejects any instance whose payload was seen before (leakage rejection), so
  the *idea of a fresh test* is enforced structurally. This is the hypothesis in the task, confirmed:
  the relation precedes the datum.
- **Honest stubs — the contract precedes the implementation.** `engine/src/bridge.rs:206-231,348`
  declares a `feature = "gpu"` module whose adapter is *"an honest stub returning Err"* with
  *"unit-typed placeholders for device/queue so the signature documents"* the future contract, plus
  an upgrade trigger. `bebop2/proto-wire/src/iroh_transport.rs:1-25` records that the old
  `unimplemented!()` iroh placeholder (an idea, `MESH-09`) has now *manifested* as a real `quinn` QUIC
  carrier — and the module doc was rewritten to match the code (spec-driven: the deviation was written
  back into the doc, per Precedent A). In both, the **idea exists as a typed, greppable contract with
  a named trigger before the code exists** — Mentalism working as designed.

**Empirical refinement (do not accept the hypothesis uncritically).** The task's stronger claim —
"an ADR *precedes* implementation" — is **falsified by the actual git history** for at least the
security ADRs. `git log --diff-filter=A` shows `docs/adr/0007-self-certifying-node-identity.md`,
`0008-local-sqlite-pq-at-rest.md`, and `0009-ssrf-safe-ip-canonicalization.md` were **all first
committed together** in `bd86908ad` (2026-07-13), whose message is *"port security **lessons** to
ADRs."* The ADRs were written *after* the code, retroactively. So the honest form of the principle
here is: **the idea must be recorded before the knowledge is lost — not necessarily before the code.**
The codebase practices documentation-of-record, sometimes retroactive, not always spec-first. This
matters for §3.

## 2. Audit methodology

Two failure directions were searched, both `file:line`-verified live, never trusted from memory:

1. **Code without idea** ("undocumented magic"): `grep -rnE '// ?(TODO|FIXME|XXX|HACK)|unimplemented!|
   todo!\(|panic!("(not|un)'` across `kernel/src`, `engine/src`, `tools`, and `bebop-repo/{bebop2,crates}`;
   a per-module `pub fn` count vs `///`/`//!` doc-line count to find logic with no recorded WHY; spot
   reads of the lowest-doc-ratio modules; a scan for stray artifacts.
2. **Idea without code** (aspirational-only claims): extracted every `*.rs` path token from
   `ARCHITECTURE.md` + five key blueprints and checked each resolves live; verified named ADRs exist;
   cross-checked concrete claims (TokenBucket, Dijkstra/A*, Union-Find/MST, `pq_kem.rs`, `matcher.rs`,
   ML-DSA-65) that `ARCHITECTURE.md` cites as *grounding*; checked whether load-bearing memory numbers
   (`recall@5=1.0`) have backing code.

The already-known instances (BRAIN-TOPOLOGY's "self-certification = claim replaces check"; the stale
`transport_policy.rs TokenBucket` cite) were **excluded** and re-checked: the TokenBucket in fact
exists (`bebop2/proto-wire/src/transport_policy.rs:30`), so that specific stale-cite is now resolved.
New instances only are reported below.

**Baseline — the codebase is unusually compliant.** dowiz `kernel/src` + `engine/src` contain **zero**
`TODO`/`FIXME`/`unimplemented!()`/`todo!()` markers (verified: `grep -rcE … | grep -v ':0'` → empty).
**Every** kernel module (39/39) and engine module (10/10) carries a module-level `//!` doc. The single
`innovate:` marker (`kernel/src/hydra.rs:704`) states its purpose. `ARCHITECTURE.md`'s technical
grounding claims all resolve: `pq_kem.rs`, `matcher.rs` exist; ML-DSA-65 is wired
(`proto-cap/src/signed_frame.rs:34-38,92`); Dijkstra/A* is real (`crates/bebop/src/cost_estimate.rs:238`);
union-find is real (`kernel/src/order_machine.rs:621`); `recall_at_k` is a real, tested function
(`kernel/src/lib.rs:140`, `verify_retrieval.rs:86-113`). Mentalism is honored well. The violations
below are the exceptions, and they cluster in the *documentation/decision-record* layer, not the code.

## 3. Findings

### F1 — ADR-020 does not exist, yet 15 docs + MANIFESTO cite it as authority — **HIGH, idea without code**
- **Evidence:** `find /root/dowiz /root/bebop-repo -iname '*ADR-020*'` → **0 results**. `docs/adr/`
  holds `0001`–`0009` + named ADRs, **no `020`**. Yet `grep -rln 'ADR-020' docs/ MANIFESTO.md` → **15
  files**, including `MANIFESTO.md` (C6). `MEMORY.md` itself lists it as *"FINAL goal: open-source
  ADR-020 (gated)."* `BLUEPRINT-P02-canon-repair-operator-decisions.md:298` independently confirms:
  *"no `docs/adr/ADR-020*.md` exists (verified: find → 0)."*
- **Why load-bearing:** ADR-020 is cited as the authoritative decision record for the **entire
  open-source / license / trademark / DCO strategy** — the thing that gates going public. The whole
  OSS arc rests on a decision record that was never written. This is the exact "claim replaces check"
  failure BRAIN-TOPOLOGY names, now instantiated on the single most consequential governance decision.
  The repo has *self-caught* it (P02) but the artifact still does not exist — the flag became another
  doc, not the missing ADR. Severity HIGH: an irreversible public action is gated on a phantom.

### F2 — Two canon blueprints disagree on where `eqc-proofs/lambda_max_of_d.rs` lives; neither states the truth — **MEDIUM, idea without code (in the cited location)**
- **Evidence:** `SYNTHESIZED-BLUEPRINT-PLAN-2026-07-16.md:174-175` cites *"the existing
  `eqc-proofs/lambda_max_of_d.rs` pattern"* as precedent an implementer should mirror, in a dowiz
  planning context. `BLUEPRINT-P02-canon-repair…:288` counter-asserts the file *"does not exist
  anywhere in dowiz."* **Ground truth:** it exists — but in **`/root/bebop-repo/rust-core/eqc-proofs/
  lambda_max_of_d.rs`**, not dowiz (`find` confirms; dowiz has `tools/eqc/` the *generator*, bebop has
  the *emitted proofs*). So SYNTHESIZED is wrong (it is not a dowiz pattern), and P02's generalization
  is imprecise (it *does* exist, in the sibling repo). A reader of SYNTHESIZED would search dowiz, find
  nothing, and be blocked.
- **Anu failure:** a decision ("mirror this existing pattern") asserted but not derivable from the
  evidence in front of the author — the file's real location was never checked. Severity MEDIUM: the
  proof itself is real and correct; only its *home* is mis-cited, but the mis-cite actively misleads.

### F3 — Retroactive ADR-writing partially inverts spec-first — **MEDIUM, systemic, idea-after-code**
- **Evidence:** ADRs `0007`/`0008`/`0009` share a single creating commit `bd86908ad` (2026-07-13)
  titled *"port security lessons to ADRs"* — documentation authored after the implementation it
  describes. The `eqc` tool (`tools/eqc/eqc.py`, `c7c1e0f5`, 2026-07-14) *did* precede its extended
  bebop proofs (`8006d53`, 2026-07-15), so the practice is mixed, not uniform.
- **Why it matters:** the stated principle (Precedent A, "the plan is the spec") implies idea-first;
  the git record shows at least the security decisions were idea-*after*. This is not fraud — a
  retroactive ADR still captures WHY before it is lost — but it is the "append-only law bias" pattern
  BRAIN-TOPOLOGY flagged (records grow by dated append, rarely reconcile), and it means the
  spec-first claim should be stated as *record-of-decision*, not *precede-implementation*. Severity
  MEDIUM because it is systemic (a habit, not a point defect) and it silently weakens the guarantee an
  ADR is supposed to give: that the design was reasoned *before* being built.

### F4 — Stray zero-byte artifact `kernel/=5` — **LOW, artifact without idea**
- **Evidence:** `kernel/=5`, 0 bytes, dated 2026-07-13, **untracked** (`git ls-files --error-unmatch`
  → fails). Almost certainly an accidental shell redirect (e.g. `… 2>=5`). It is a literal artifact
  with no idea behind it — nobody recorded why it exists, because it is accidental cruft.
- **Severity LOW:** untracked, zero-byte, zero blast radius. Included because it is the cleanest
  concrete instance of "manifestation with no pattern" and it is trivially deletable (`rm 'kernel/=5'`).

### Non-findings (checked, cleared)
- `transport_policy.rs TokenBucket` — **exists** (`bebop2/proto-wire/src/transport_policy.rs:30`); the
  prior stale-cite is resolved.
- `ARCHITECTURE.md` grounding docs (`SYNTHESIZED…`, `BRAIN-TOPOLOGY…`, `STRATEGIC-VECTORS…`) — now
  **git-tracked** (the session-start `??` snapshot was stale); "canon cites uncommitted files" does
  not hold.
- `ARCHITECTURE.md` F45/F46 (Dijkstra/A*, Union-Find/MST for *mesh-heal*) — asserted with an explicit
  `(P1) NOW: implement` gap tag. This is Mentalism working *correctly*: the idea is recorded and
  **honestly marked as not-yet-manifested**, which is the sanctioned form, not a violation.

## 4. Verdict

**This codebase honors Mentalism strongly at the code layer and weakly at the decision-record layer.**

The Rust core is a near-model implementation of the principle: zero unimplemented markers, universal
module documentation, a literal equation-compiler (`eqc`) whose output is a proof-carrying derived
artifact, a metamorphic generator whose instances are defined by relation before data, and honest
typed stubs with named upgrade triggers. Where the code embodies an idea, the idea is almost always
written down and often mechanically checkable. On the two failure axes searched, "code without idea"
is essentially absent (one accidental zero-byte file, F4).

The failures are all "idea without code" and they live in the *governance / decision-record* layer:
a phantom ADR-020 that the entire OSS strategy cites but nobody wrote (F1, HIGH); two canon blueprints
that disagree about a file's location, neither correct (F2, MEDIUM); and a documented habit of writing
ADRs *after* the code they justify (F3, MEDIUM). These are precisely the class BRAIN-TOPOLOGY
predicted — *claim replaces check*, *append-only accretion* — and they are dangerous exactly because
the code discipline is so good that a reader trusts the docs by association. The single most important
remediation is to **make ADR-020 exist** (or delete every citation to it); a decision cited fifteen
times and manifested zero times is the largest live violation of "The All is Mind" in the repo: the
idea was *declared real* without ever being *made real*.
