7.6  WIRE FORMAT

  Transport layer (proto-wire):
    - iroh (UDP QUIC) + WSS (WebSocket over TLS) carriers
    - TLS via rustls (ring backend), explicitly NO openssl-sys
    - Plain ws:// for testing, wss:// via tokio-rustls for production
    - Channel binding via SHA3-256 handshake transcript
    - QUIC via quinn crate (Ring-based)

  Capability layer (proto-cap):
    - Per-frame signed capabilities (replaces JWT)
    - Hybrid gate: Ed25519 (classical) + ML-DSA-65 (PQ), BOTH enforced
    - TLV codec on signing path (NEVER serde for signing — serde is non-canonical)
    - Scope-constrained with typed scope definitions
    - Revocation support via roster + claim machine
    - Signed frame envelope: TLV-encoded, SHA-256 body digest
    - Capability frame transported as `x-dowiz-cap` HTTP header (base64)

  Crypto verification ladder (proto-crypto):
    - SCAFFOLD: Wycheproof -> FIPS re-derivation pipeline (not yet implemented)
    - Constant-time marker for critical paths
    - Intended to layer formal verification over bebop2-core primitives
