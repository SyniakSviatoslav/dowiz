C. PATTERNS / INVARIANTS / IDEMPOTENCY NOTES
============================================================================

  1. ZERO-DEP DISCIPLINE: Both kernel and engine have verified zero-external-
     crate default builds (`cargo tree -e no-dev` confirms). All external
     crates are behind feature flags (serde, gpu, text, etc.), keeping default
     build offline-clean.

  2. COMPILE FIREWALL: agent-loop must never import kernel. Enforced by:
     (a) source grep in adversarial test, (b) Cargo.toml grep in CI.
     Confirmed clean.

  3. CPU FLOOR IS AUTHORITATIVE: GPU is a display surface only. Authoritative
     compute is CPU-side so scalar==SIMD is bit-identical across platforms.
     Engine's `field_frame`, `clock`, `sdf` are pure CPU.

  4. FEATURE-GATING DISCIPLINE: Every external dependency feature follows the
     same pattern: declare the feature, gate the module behind
     `#[cfg(feature = "...")]`, stub returns Err if feature absent. Applied
     consistently across kernel, engine, agent-adapters, llm-adapters.

  5. NO TOKIO IN DEFAULT BUILD: llm-adapters uses ureq (sync) for HTTP,
     agent-adapters uses no tokio, kernel uses tokio only behind `pgrust`
     feature. All default builds are sync/std-only.

  6. WASM SURFACE PATTERN: Both wasm crates use `cdylib + rlib` crate-type,
     allowing native `cargo test` and WASM `cargo build --target wasm32`.

  7. COURIER IS PURE CONSUMER: apps/courier re-owns none of kernel/engine
     logic. It consumes wire frames and mirrors P65 dispatch events locally.

  8. NO WORKSPACE: All crates are standalone (sibling to kernel/), NOT
     workspace members. Each has its own Cargo.lock.

  9. RUNTIME WASM BINDING: web/ manually re-implements wasm-bindgen's glue
     layer in `kernel_client.mjs` (no generated bindings in the repo), synced
     against wasm-bindgen 0.2.126 output conventions.

 10. RUST-ONLY SURFACES: Both courier and engine are explicit: "No DOM, no
     webview, no JS" / "no TS/JS runtime deps." TS/JS is the legacy oracle.
