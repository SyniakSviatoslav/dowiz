### The worst money/checkout localization bug (answer to the brief)

**P0 — `apps/web/src/pages/admin/PromotionsPage.tsx:360,380` — a 100× display-vs-charge money lie.**
Money is stored/charged in **whole lek** (`CURRENCIES.ALL.decimals = 0`; `formatMoney` renders the raw integer; the promo form saves `parseInt` raw; the server applies `promo.discount_value` directly against the lek subtotal at `apps/api/src/routes/owner/promotions.ts:209-211`). But the PromotionsPage list divides by 100 for display: `(discount_value/100).toFixed(0)} ALL` and `min_order_amount/100`. So a promo stored & charged as **500 ALL displays as "5 ALL"**. An owner who "fixes" the display by typing 50000 will silently give away **50,000 lek per order** at checkout. This is a display-vs-charge divergence on money — it should gate before any real owner creates a promo. (Strictly a units bug that manifests through the money-formatting path; it is the single most dangerous money-facing defect found.)

The runner-up localization bug — the one that is purely l10n — is `packages/shared-types/src/utils.ts:26`: the shared `formatMoney` ignores its own `CURRENCIES.ALL` config (`locale: 'sq-AL'`, `symbol: 'L'`) and hardcodes `` `${Math.round(amount)} ALL` `` — no thousands grouping ("12500 ALL" instead of "12 500 L"), ISO code instead of the Lek symbol, on **every price surface in the app** (cart, checkout, order status, admin). The EUR path (`:40`) hardcodes `toFixed(2) + ' €'` with an English dot decimal, so sq/uk users in EUR mode see "4.50 €" instead of "4,50 €".

## Root causes (fix these 4 utilities and ~40 line-items collapse)

1. **No plural machinery.** `packages/ui/src/lib/i18n.ts:26-45` `t()` does flat `{{k}}` string substitution only. Zero `Intl.PluralRules` anywhere in `apps/web` or `packages/ui`. Ukrainian's 3 forms (1 / 2-4 / 5+) and Albanian's singular are structurally unrenderable → every plural finding below.
2. **No shared locale-aware date formatter.** The one correct pattern (`LOCALE_MAP` → BCP-47, `apps/web/src/pages/courier/HistoryPage.tsx:40`) is used in exactly one file. 10 other call sites use bare `.toLocaleDateString()/.toLocaleString()` (browser locale wins, ignores the in-app switch) or hardcode `'en-GB'`.
3. **Locale-blind `formatMoney`** (`packages/shared-types/src/utils.ts:20-41`) — see above.
4. **No focus management in the shared dialog primitives** (`ResponsiveDialog.tsx`, `ConfirmDialog.tsx`) — no dialog in the codebase moves focus in, traps Tab, or restores focus on close. Fixing the two primitives repairs every modal on every surface.
