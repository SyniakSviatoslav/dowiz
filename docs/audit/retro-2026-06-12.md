## 6 failure patterns → 6 countermeasures

### Pattern 1: Hardcoded infrastructure URLs with no validation
`cdn.dowiz.org` was hardcoded in 13 source files but never configured in Cloudflare.
**Countermeasure:** Add `no-hardcoded-env-url` lint rule that flags absolute URLs pointing to project's own domains (cdn, api) — they must come from env or config.

### Pattern 2: Gate blind to client-side rendering
B1-B6 tested backend health, migrations, workers, E2E order flow — but never opened a browser.
**Countermeasure:** Already done — B7 (menu content) + B8 (asset loading) added to release-gate.ts.

### Pattern 3: No a11y enforcement
The `e2e/helpers/a11y.ts` file exists with working checks (axe-core, touch targets, form labels, aria-live) but NOTHING calls them.
**Countermeasure:** Wire `checkAxe()` + `checkTouchTargets()` + `checkFormLabels()` into the E2E smoke spec (`live-smoke.spec.ts` or `deploy-validation.spec.ts`).
Also: install `eslint-plugin-jsx-a11y` and enable recommended rules.

### Pattern 4: No i18n key coverage check
Keys were missing from UK locale (15+), hardcoded text existed in alerts, nutrition labels, status labels, dropoff options. No automated check verifies all `t()` calls have matching keys in all 3 locales.
**Countermeasure:** Write an i18n coverage script (or add to `verify:all`) that:
- Extracts all `t('key'` patterns from source
- Cross-references against `messages.sq`, `messages.en`, `messages.uk`
- FAILS if any key is missing from any locale

### Pattern 5: No hardcoded-text lint rule
`alert('Only image files')`, `alert('Max 5 MB')`, hardcoded nutrition labels — all bypassed detection.
**Countermeasure:** Add `local/no-hardcoded-string-in-ui` lint rule that flags `alert('...')` and similar patterns in UI packages — must use `t()`.

### Pattern 6: No design-token audit in gate
Contrast violations (35), touch targets (110) — no automated check.
**Countermeasure:** Add contrast ratio check script to `verify:all` that validates all `--brand-text-muted` + `--brand-primary` combos against WCAG AA 4.5:1. Add to B7/B8 gate or pre-deploy checklist.

## Specific action items

### 1. ESLint: add `local/no-hardcoded-string` rule
File: `tools/eslint-plugin-local/src/index.js`
```js
// Flags alert('...') and similar in UI packages
// Must use t('key', 'fallback') instead
'local/no-hardcoded-string': 'warn'
```
Scope: `apps/web/src/**, packages/ui/src/**`

### 2. ESLint: install `eslint-plugin-jsx-a11y`
```bash
pnpm add -D eslint-plugin-jsx-a11y -w
```
Enable in `eslint.config.js`:
```js
import jsxA11y from 'eslint-plugin-jsx-a11y';
// ... in config array:
jsxA11y.flatConfigs.recommended,
```

### 3. E2E: wire a11y helpers into smoke test
In `e2e/tests/live-smoke.spec.ts` (or `deploy-validation.spec.ts`), add:
```ts
import { checkAxe, checkTouchTargets, checkFormLabels } from '../helpers/a11y.js';
test('menu page has no critical a11y violations', async () => {
  await page.goto('/s/demo');
  const violations = await checkAxe(page);
  expect(violations.filter(v => v.impact === 'critical').length).toBe(0);
});
```

### 4. i18n coverage script
New script at `apps/api/scripts/verify-i18n-coverage.ts` (or add to `packages/ui`):
- Parse `i18n.ts` to extract all keys from `sq`
- Parse all `apps/web/src/` + `packages/ui/src/` for `t('key'` patterns
- Report missing keys (used in code but not in i18n.ts)
- Report keys present in `sq` but missing from `uk` or `en`
- Exit 1 on any missing key

### 5. Contrast audit script
New step in `verify:launch` or gate: compute WCAG contrast ratio for `--brand-text-muted` on `--brand-surface` and `--brand-primary` on `--brand-bg` for each of the 6 presets. Fail if any < 4.5:1 for normal text.

### 6. Update `verify:all` composite
Add to the composite chain:
```
env → db → rls → secrets → migrations → lint → typecheck → i18n-coverage → a11y-lint → contrast-audit
```
