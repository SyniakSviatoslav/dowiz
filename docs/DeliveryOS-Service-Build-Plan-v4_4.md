## 1. Місія та явні межі скоупу (новий кадр)

**Місія dowiz:** локалізована й анонімізована **власна** доставка для закладу. Вирішуємо **конкретну** проблему — не «все підряд».

- **Поза скоупом за рішенням:** генерація попиту/відкриття закладу. dowiz **конвертує наявний попит**, а не створює його — це задача закладу, не платформи. SEO/JSON-LD/Google-Business-лінк лишаються (вони *конвертують* наявну видимість), але це не місія й не вкладаємось понад потрібне.
- **Клин проти звички до агрегатора — ціна.** Відсутність маркетплейс-комісії = дешевше клієнту й більша маржа власнику. Економіка б'є звичку.
- **Перші клієнти мають власних кур'єрів** (валідовано). Додатково: навіть **без кур'єра** зручні **pre-order (scheduled)**, **pickup** і **бронювання** → закладаємо як шви схеми зараз, рантайм пізніше.
- **Соц/маркетинг-автоматизації відкладені.** Виняток: **Telegram як операційний канал сповіщення власника** — це не «соц-бот», а несуча ланка надійності (§6).

## 2. Реєстр нових рішень (закрито — не передоговорювати)

| Тема | Рішення |
|---|---|
| Модифікатори меню | **Робимо зараз.** Групи опцій (`min/max/required`) + опції з `price_delta`; перевикористовувані між продуктами (M:N). |
| Custom properties | **Обидва типи:** структуровані платні модифікатори **+** описові `attributes jsonb` (алергени, теги, гострота). |
| Поверхні | **3 з одного движка:** хостована сторінка-PWA (`/s/:slug`) · `iframe` (`?embed=1`, postMessage-height) · drop-in `widget.js` (SRI+версіонування, оверлей). |
| Брендування | **Лого + колір + шрифт** (курований набір, перевірка албанських гліфів ë/ç). Кастомний домен — **пізніше** (поки піддомен `slug.dowiz.org`). |
| i18n | **Перемикання мови на клієнті** (усі переклади в одному payload → кеш лишається на `menu_version`). Багато мов. |
| Контент-переклад | **Авто-переклад** контенту меню (через menu-only ШІ-шлях, §3). |
| Захоплення локації | **Пін на карті — основний спосіб** (мітигація слабкої адресації в Албанії), не парс адреси. |
| Дрейф кошика | Явний флоу **«меню оновилось — перегляньте кошик»** + грейсфул-реджект стоп-лист-позиції/модифікатора. |
| Округлення грошей | **Політика округлення визначена** (для відсоткових промо/ПДВ; integer ALL лишається). |
| Меню-імпорт | **Мінімальний ШІ-імпорт — у Фазу 2 CORE** (онбординг-friction критичний), не fast-follow. |
| Видимість | SEO + JSON-LD (`Restaurant`/`Menu`) + Google-Business «Order online»-лінк — у Фазу 2 CORE (конвертація, не генерація). |
| Кур'єри | Власний флот (перші клієнти). Контракт `deliver` — Фаза 3, проєктуємо на папері паралельно. |
| «Генерація» (ідея власника) | **Запарковано, не MVP.** |

## 3. Інваріант даних і ШІ (governance) — новий, жорсткий

> **Жоден PII власника/кур'єра/клієнта не йде в жодну модель без явної попередньої згоди.**

- **ШІ торкається ВИКЛЮЧНО контенту меню** — назви/описи/ціни позицій та їх переклад. Нічого більше.
- **Шов `MenuParserProvider`** (та сама dependency-inversion, що інші шви). Безкоштовний шлях: **PaddleOCR/Tesseract** (open-source, CPU, self-host) витягає текст із фото/PDF → **безкоштовний текстовий LLM** структурує в наш канон. Зображення — лише меню, **ніколи** клієнтські дані.
- **Обов'язковий ручний перегляд** власником перед commit (ШІ пропонує, власник володіє). Це понижує планку моделі до «зекономити друкування» — тому безкоштовні моделі життєздатні.
- **Авто-переклад меню** йде тим самим menu-only шляхом (дозволено — це контент меню, не PII).
- **Модель свопиться за швом** під юрисдикцію/приватність (уникати free-tier, що тренується на даних, для будь-чого чутливого; Gemini free, напр., тренується на вхідних даних і виключає комерційне використання в EU — прийнятно лише для меню, не для PII).

## 4. Шви в схему — закласти ЗАРАЗ (нульова рантайм-вага, RLS з моменту створення)

Доктрина v3.1: «схема повна, код ні». Розширюємо до коду Фази 2 — це найдешевше вікно; ретроспективна міграція money/menu-таблиць болить. **Закладаємо максимум; рантайм не розбухуємо.**

**Модифікатори (робимо рантайм зараз):**
- `modifier_groups` (`name`, `min_select`, `max_select`, `required`)
- `modifiers` (`group_id`, `name`, `price_delta integer CHECK(>=0)`, `available bool`)
- `product_modifier_groups` (M:N, `sort_order`)
- `order_item_modifiers` (снапшот: `name_snapshot`, `price_delta_snapshot`, nullable FK)

**Контент-i18n (схема зараз, UI стартує al/en):**
- `product_translations(product_id, locale, name, description)` + `category_translations`
- `locations.default_locale`, `locations.supported_locales text[]`

**Описові атрибути:** `products.attributes jsonb`

**Гроші-розбивка (критично — money-таблиця, щоб не мігрувати потім):**
- `orders.delivery_fee`, `orders.discount_total`, `orders.tax_total` — `integer NOT NULL DEFAULT 0`
- `total = subtotal + delivery_fee + tax_total − discount_total` (серверний розрахунок; **політика округлення задокументована**)

**Валюта (прихований регіональний баг):** `locations.currency_code` (ISO-4217, default `ALL`) + `currency_minor_unit`

**ПДВ:** `locations.tax_rate`, `locations.price_includes_tax bool` (TVSH 20%)

**Доставкова ціна:** конфіг-поля локації (`min_order_value`, `free_delivery_threshold`, `delivery_fee_flat`) + `delivery_tiers(location_id, max_distance_km, fee, min_order?)` + `delivery_polygon jsonb` (scaffold) + `delivery_radius_km` (є)

**Зображення:** `products.image_key` (Supabase Storage)

**Лог переходів:** `order_status_history` (append-only — узгоджено з «жоден стан не пастка», безцінно для дебагу проду соло)

**Pre-order / pickup / scheduled:** enums scaffold є; `orders.scheduled_at`, `orders.pickup_code` є — лишаємо (рантайм пізніше)

**Бронювання (новий шов):** `reservations(id, location_id, customer_id, party_or_slot, slot_at, status)` — scaffold, рантайм пізніше

**Лояльність/повторний клієнт (легкий шов):** `customers.marketing_opt_in`, `customers.loyalty_points`

> **Дисципліна:** рантайм **НЕ** будуємо для scheduled/pickup/booking/loyalty/polygon/distance-понад-flat зараз. Лише схема.

## 5. Перепріоритезована Фаза 2

### CORE (будуємо; якість і надійність не чіпаємо)

1. **Міграція-розширення** (усі шви §4) — *перший build-промпт, усе решта на ньому.*
2. **Меню-CRUD** + `menu_version++` на мутаціях + стоп-лист (продукт **і** модифікатор) + i18n-каркас.
3. **Ціновий движок:** модифікатори + доставка + округлення; `request_hash` канонізується з **відсортованими** `modifier_ids` (інакше ідемпотентність протече). Серверна ціна, нуль довіри клієнту.
4. **Меню-імпорт:** CSV (детерміновано) **+ мінімальний ШІ-імпорт** (PaddleOCR→free-LLM→канон→**ручний перегляд**→атомарний commit). Онбординг-критично.
5. **Авто-переклад** контенту меню (menu-only ШІ).
6. **SSR `/s/:slug`** + Cloudflare-кеш по `menu_version` (обов'язково — публічна сторінка = DoS-поверхня на N=1) + **JSON-LD** (`Restaurant`/`Menu`/`MenuItem`/`LocalBusiness`) + `hreflang` + інструкція власнику по Google-Business «Order online»-лінку.
7. **Клієнтський флоу:** меню→кошик(із модифікаторами)→checkout(**пін на карті**, зона/distance-комісія на сервері)→статус; pre-fill із localStorage; **флоу дрейфу кошика**; грейсфул стоп-лист-реджект. **PWA-маніфест per-location** (ім'я/іконка/колір закладу → «додаток ресторану»).
8. **Брендування** (лого+колір+шрифт → CDN, нуль хардкоду) + **3 поверхні** (PWA · `iframe?embed` · `widget.js` SRI+версіонування) + піддомен `slug.dowiz.org`.
9. **Шов сповіщень + ранній Telegram-ops** (§6) — попри початкову «Фазу 4», надійність алерту власнику цього вимагає; шов + Telegram заводимо рано.

### Відкладене (рантайм пізніше; схема вже готова §4)

- Scheduled/pre-order + pickup + бронювання — рантайм.
- Distance-комісія, якщо геокод/пін по Албанії хитає (старт: flat + min-order + free-threshold).
- Кастомні домени (Cloudflare custom hostnames — має ціну на масштабі).
- Лояльність, полігон-зони, промо — рантайм.
- WhatsApp/Viber/SMS-адаптери сповіщень (після Telegram, §6).
- Ідея «генерації» — поза MVP.

## 6. Стратегія сповіщень (Albania-aware, через шов `NotificationProvider`)

Кожен канал — адаптер за швом. Послідовність — за **співвідношенням охоплення й вартості інтеграції** для соло-розробника:

1. **Native push** — FCM (Android-PWA) / APNs (iOS) — in-app шар. *Засторога: iOS-PWA-push історично крихкий (лише для встановленого PWA) — тому не єдина ланка.*
2. **Telegram-ops — перший fallback.** Bot API безкоштовний, миттєвий, без апруву → ідеальний для соло. **Несуча ланка надійності алерту власнику** рано.
3. **WhatsApp** — імовірно найвживаніший в Албанії (регіональний + глобальний патерн: WhatsApp #1 у 70/100 країн, ~84% пристроїв, ~98% open-rate). Але Business API потребує Meta-верифікації + BSP + шаблони + плата за діалог → додаємо, коли трафік виправдає; довгостроково — основний канал власника.
4. **Viber** — сильний у Південно-Східній Європі/на Балканах (#1 у Сербії/Болгарії/Греції); має Business/bot API → розумний «балканський» додаток після Telegram.
5. **SMS** — універсальний last-resort (без застосунку), плата за повідомлення + шлюз.

> **Чесно:** прямих Albania-специфічних розкладок у джерелах не знайшлось; висновок — з регіонального патерну ПСЄ (WhatsApp-дефолт + сильний Viber) + вартості інтеграції. **Перевірити прямо в перших ~50 власників: яким месенджером користуються щодня.**

## 7. Реєстр ризиків — доповнення (нове, з мітигаціями)

| Ризик | Статус / мітигація |
|---|---|
| Слабка адресація/геокод в Албанії ламає distance-ціни й зони | **Пін на карті — основний** спосіб локації; distance — лише коли пін надійний |
| Дрейф «кошик vs `menu_version`» | Явний флоу перегляду кошика + грейсфул-реджект |
| Фрагментація SSR-кешу (N мов × бренд-CSS) | Переклади на клієнті → ключ лишається `menu_version` |
| Integer-гроші + відсотки | Задокументована політика округлення; поля `discount_total/tax_total` |
| Надійність алерту власнику (iOS-PWA-push крихкий) | Telegram-ops рано + багатоканальний шов |
| Соло-розробник, нема команди підтримки | Автоматизація: тести + health-чеки + винятковий self-serve/error-UX (прийнято) |
| Friction онбордингу | Мінімальний ШІ-імпорт у CORE |
| Багатомовність порожня без перекладу | Авто-переклад меню (menu-only ШІ) |
| Скоуп-creep проти MVP-дисципліни | **Керівне правило: схема багата, рантайм безжально мінімальний** |
| Звичка до агрегатора | Клин — ціна (економія обом сторонам); прийнято як позиціювання |

**Перенесені з попередніх документів (без змін):** N=1 = повний простій при сплеску/краші/деплої; готівковий облік на чесному вводі; фрикшнлес-клієнт піднімає ризик фейків (закрито шарами тертя); людський SPOF на N=1.
